## 2024-05-18 - Optimized Object.values() in pure Zustand selectors
**Learning:** `Object.values(state.entities.projects)` creates a new array reference every time the selector runs, even if the raw dictionary object hasn't changed. In lightweight selector setups without `reselect`, this breaks `useShallow` checks and causes expensive cascading React re-renders throughout the application.
**Action:** Implemented module-level closures to cache `lastRaw` and `lastArray` in `src/store/selectors.ts` for major entity collections (`projects`, `talents`, `rivals`, `activeProjects`, `releasedProjects`). This ensures identical array references are returned when the source dictionary is unchanged, turning O(N) allocations into O(1) cache hits and significantly reducing garbage collection pressure.
